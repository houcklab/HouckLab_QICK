QuarkTab Class¶

QuarkyTab.py¶

The custom QQuarkTab class for the central tabs module of the main application.

Each QQuarkTab is either an experiment tab or a data tab that stores its own object attributes, configuration, data, and plotting. Arguable is more important for functionality than the main Quarky.py file.

class scripts.QuarkTab.QQuarkTab(experiment_path=None, tab_name=None, is_experiment=None, dataset_file=None)¶

Bases: QWidget

The class for QQuarkTabs that make up the central tabular module that hold specific experiments or datasets.

Important Attributes:

  • experiment_obj (Experiment Module): The experiment module object that was passed.

  • config (dict): The configuration of the QQuarkTab experiment/dataset.

  • data (dict): The data of the QQuarkTab experiment/dataset.

  • plots (pyqtgraph.PlotWidget[]): Array of the pyqtgraph plots of the data.

  • plot_widget (pyqtgraph.GraphicsLayoutWidget): The graphics layout of the plotting area

__init__(experiment_path=None, tab_name=None, is_experiment=None, dataset_file=None)¶

Initializes an instance of a QQuarkTab widget that will either be of type experiment based on the parameters passed.

  • An experiment will pass: [experiment_path, tab_name, is_experiment=True].

  • A dataset will pass: [tab_name, is_experiment=False, dataset_file]

Parameters:

  • experiment_obj (Experiment Module) – The experiment module object extracted from an experiment file.

  • tab_name (str) – The name of the tab widget.

  • is_experiment (bool) – Whether the tab corresponds to an experiment or dataset.

  • dataset_file (str) – The path to the dataset file.

auto_plot_prepare()¶

Automatically prepares the data based on its shape. This is not always correct but attempts to infer. This method can be helpful when writing a custom plotter function. Works for both loading data as well as experiment data.

The prepared data will be in the format:

{
    "plots": [
        {"x": x_data,
        "y": y_data,
        "label": name,
        "xlabel": "Qubit Frequency (GHz)",
        "ylabel": "a.u."},
    ],
    "images": [
        {"data": data,
        "label": name,
        "xlabel": "X-axis",
        "ylabel": "Y-axis",
        "colormap": "inferno"},
    ],
    "columns": [
        {"data": data,
        "label": name,
        "xlabel": "X-axis",
        "ylabel": "Y-axis"},
    ]
}
capture_plot_to_clipboard()¶

Captures a screenshot of the plot via a QPixmap and saves it to the users clipboard. Paste normally.

change_output_dir()¶

Changes the output directory that data is autosaved to via a file dialog.

clear_plots()¶

Clears the plots.

export_data()¶

Is the function called when the export button clicked. Prepares file naming and saves data.

load_dataset_file(dataset_file)¶

Takes the dataset file and loads the dict, before calling the plotter.

Parameters:

dataset_file (str) – The path to the dataset file.

plot_data()¶

Plots the data of the QQuarkTab experiment/dataset using prepared data that is prepared by the specified plotting method of the dropdown menu.

prepare_file_naming()¶

Prepares naming conventions by creating folder and file names with experiment type and timestamps.

process_data(data)¶

Processes the dataset usually in the form of averaging.

Parameters:

data (dict) – The data to be processed.

reload_experiment()¶

Reloads the current experiment via the experiment_path given. This allows a user to change the experiment code or even the plotter without having to exit the GUI, close any tabs, or import the experiment from scratch.

remove_plot()¶

Retrieves the cursor’s current position and removes the corresponding plot that it is hovering over from the layout.

save_data()¶

[UNCOMPLETE]

Saves data by force dumping it into an h5 format. Works as is but doesn’t save metadata that could be important for sweeps and configurations.

setup_plotter_options()¶

This method sets up the plotting options based on the current experiment. If the experiment gave a plotter function, it is automatically set as the default method. Otherwise, the current methods available only include ‘Auto’.

setup_signals()¶

Sets up all the signals and slots of the QQuarkTab widget. This includes the toolbar buttons, plot functionalities, and export settings.

update_coordinates(pos)¶

Updates the coordinates label to reflect the cursor’s location on a plot’s axis.

Parameters:

pos (tuple) – The coordinates of the cursor

update_data(data)¶

Is the slot for the emission of data from the experiment thread. Calls the methods to process and plot the data.

Parameters:

data (dict) – The data to be processed.
