AccountsPanel Class¶

AccountsPanel.py¶

The sidepanel that saves RFSoC connection information in different accounts for connection made easy.

Creates a local dataset of accounts in an accounts folder within the Quarky_GUI directory and stores your preferred default connection.

Each account is saved locally in a json file named after the account’s name that it stores. Format:

{ "ip_address": "111.111.1.111", "account_name": "houcklab" }

The accounts folder will also always have a file named default.json that stores the account name of the default account. Format:

{"default_account_name": "houcklab"}
class scripts.AccountsPanel.QAccountPanel(parent=None)¶

Bases: QWidget

A custom QWidget class for the entire accounts panel.

Important Attributes:

  • accounts_list (QListWidget): The QListWidget item that lists all the accounts saved locally

  • form_layout (QFormLayout): The QFormLayout layout that has all relevant fields for editing/saving an account.

  • current_account_item (QListWidgetItem): Stores the currently selected account item from the accounts list.

__init__(parent=None)¶

Initializes an instance of this AccountsPanel widget (custom QWidget) class.

Parameters:

parent (QWidget) – The parent QWidget that this widget belongs to.

attempt_connection_or_disconnect()¶

Attempts to connect to an account or disconnect it based on whether or not the variable connected_account_name is None or not. In both cases, it emits the corresponding signal to the main GUI telling it how to handle the RFSoC.

create_account()¶

Creates a new account by populating a new json file per the format listed at the top. Retrieves the text within the form boxes and validates them before doing so. Then, it creates a new QListWidgetItem and adds it to the accounts_list.

delete_account()¶

Deletes the selected account from the accounts_list and removes the corresponding json file. Prevents default account deletion.

disable_since_connected()¶
A UI function that handles the case where we are actively connected to an RFSoC.

  • Allow: Disconnecting

  • Disallow: Form editing, Saving, Deleting, Creating, Setting as Default

load_accounts()¶

This function populates accounts_list QListWidget with all the accounts locally saved in the accounts directory. See the top for the json format for storing an account. It populates the instance variables for the current/default account names and items. It also selects the default account.

rfsoc_attempt_connection¶

The Signal sent to the main application (Quarky.py) that tells the program to attempt a connection to a RFSoC via the given IP address.

Parameters:

ip_address (str) – The IP address to attempt a connection to.

rfsoc_connection_updated(ip_address, status)¶

Upon an RFSoC connection update of either ‘failure’ or ‘success’, the main widget sends a signal to the accounts panel that is handled via this function. If success, it updates the UI and corresponding name and item variables.

Parameters:

  • ip_address (str) – The IP address of the RFSoC.

  • status (str) – The status of the RFSoC connection (‘success’ or ‘failure’).

rfsoc_disconnect¶

The Signal sent to the main application (Quarky.py) that tells the program to disconnect the RFSoC.

saved_indicate()¶
A UI function that handles the case where the form entries are not edited.

  • Allow: Setting as Default, Connecting/Disconnecting, Deleting

  • Disallow: Saving, Creating

select_item(current, previous=None)¶

The function that is called when the user selects an item from the accounts list (clicks on an QListWidgetItem. It updates the coresponding name and item variables and opens the json file to populate the form fields.

Parameters:

current (QListWidgetItem) – The selected item.

set_as_default()¶

Function triggered when the default_button is clicked. Calls the update_default_account_name function and reloads all accounts.

setup_signals()¶

Sets up all the signals and slots of the Accounts Panel. Including form editing/action buttons, RFSoC connection attempts, and accounts list selection.

unsaved_indicate()¶
A UI function that handles the case where form entries have been edited, meaning changes are currently unsaved.

  • Allow: Saving, Creating, Connecting/Disconnecting, Deleting

  • Disallow: Setting as Default

update_account()¶

Updates an account by validating the form entries and editing the respective json form fields. See the top of the documentation for json format.

update_default_account_name(new_default_account_name, new_default_account_item)¶

A function that updates the default account name within the default.json file (format at the top of the documentation) with the new passed account name. It also updates the default account list item.

Parameters:

  • new_default_account_name (str) – The new default account name.

  • new_default_account_item (QListWidgetItem) – The new default account item.

validate_account_input(new_ip_address, new_account_name, purpose)¶

Validating the input to create accounts via an account name and an ip_address. Moreover, based on the purpose of the validation (either ‘create’ or ‘update’), it will check for account name duplicates.

Parameters:

  • new_ip_address (str) – The new IP address of the account.

  • new_account_name (str) – The new account name of the account.

  • purpose (str) – The purpose of the validation, to either ‘create’ or ‘update’ an account.

Returns:

True if the input is valid, False otherwise.

Return type:

bool
